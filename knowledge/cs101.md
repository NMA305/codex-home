## What is Computer Science?
### ما هو علم الحاسوب (Computer Science)؟

علم الحاسوب هو **مجال دراسي يختص بفهم كيفية عمل الحواسيب** (الأجهزة والبرمجيات) **وتصميم حلولٍ تُنفّذ على هذه الحواسيب**. يمكن تلخيصه في ثلاثة محاور رئيسية:

| المحور | ما يعنيه | مثال بسيط |
|--------|----------|-----------|
| **الخوارزميات** | خطوات محددة لحل مشكلة ما. | طريقة حساب مجموع الأعداد من 1 إلى 100. |
| **البرمجة** | كتابة الخوارزميات بلغة يستطيع الحاسوب فهمها. | كتابة برنامج بلغة بايثون لحساب المتوسط. |
| **النظم** | بنية الحاسوب وكيفية تفاعل مكوّناته (المعالج، الذاكرة، التخزين، الشبكات). | تصميم نظام تشغيل يدير تشغيل التطبيقات. |

## لماذا يُطلق عليه "علم"؟

1. **قابلية القياس والتحليل**  
   - نستخدم الرياضيات لتقييم كفاءة الخوارزميات (مثلاً، كم من الوقت تستغرق لمعالجة بيانات معينة).

2. **مبادئ عامة**  
   - القواعد التي نتعلمها (مثل هياكل البيانات، نظرية التعقيد) تنطبق على كل أنواع الحواسيب، سواء كان هاتفًا ذكيًا أو سوبركمبيوتر.

3. **اكتشافات وبحوث**  
   - يجرى بحثاً مستمراً لتطوير خوارزميات أسرع، لغات برمجة أكثر أمانًا، أو طرق جديدة لتخزين البيانات.

## 3. مثال عملي خطوة بخطوة: **برنامج لحساب المتوسط**

### الخطوة 1: تحديد المشكلة
> نريد برنامجًا يقرأ درجات طلاب (عدد غير معروف) ثم يُظهر المتوسط.

### الخطوة 2: صياغة الخوارزمية (خطوات الحل)
1. اطلب من المستخدم إدخال عدد الدرجات `n`.  
2. كرّر `n` مرة:  
   - اطلب درجة واحدة.  
   - أضفها إلى مجموع الدرجات.  
3. احسب المتوسط = المجموع ÷ `n`.  
4. اعرض النتيجة.

### الخطوة 3: اختيار لغة برمجة
نستعمل **Python** لأنها سهلة للمبتدئين.

### الخطوة 4: كتابة الكود

```python
# برنامج لحساب المتوسط
n = int(input("كم عدد الدرجات؟ "))   # خطوة 1
total = 0                              # المتغيّر لتجميع الدرجات

for i in range(n):                     # خطوة 2
    grade = float(input(f"ادخل الدرجة {i+1}: "))
    total += grade

average = total / n                     # خطوة 3
print(f"المتوسط هو: {average:.2f}")    # خطوة 4
```

### الخطوة 5: تشغيل البرنامج
- تفتح محرّر (مثل VS Code أو حتى الـ IDLE المدمج مع Python).  
- تنسخ الكود وتنفّذه.  
- يُظهر لك البرنامج أسئلة، تُدخل الأجوبة، ثم يطبع المتوسط.

## Data structures and algorithms
## مقدّمة سريعة  
**Data Structures (هياكل البيانات)** و **Algorithms (الخوارزميات)** هما أساس أي برنامج أو نظام حاسوبي.  

- **هياكل البيانات**: طريقة تنظيم وتخزين المعلومات داخل الذاكرة لتسهيل الوصول إليها ومعالجتها.  
- **الخوارزميات**: مجموعة من الخطوات المتتابعة (قواعد) تُنفّذ على هذه البيانات لتحقيق هدف معيّن (مثل البحث، الفرز، التحديث …).

تخيل أن لديك مكتبًا لتخزين ملفاتك. كيف ترتّب الملفات (هياكل البيانات)؟ وكيف تبحث عنها أو تنقلها (الخوارزميات)؟

## 1. هياكل البيانات الأساسية

| الاسم | الوصف البسيط | مثال عملي (بلغة بايثون) | متى نستخدمه؟ |
|-------|--------------|------------------------|--------------|
| **المصفوفة (Array / List)** | مجموعة من العناصر المتجاورة ذات نفس النوع، يمكن الوصول إلى أي عنصر مباشرة عبر الفهرس. | `arr = [10, 20, 30, 40]` <br> `arr[2]  # 30` | عندما تحتاج وصولًا سريعًا (O(1)) إلى عنصر بمعرفة موقعه، وحجم البيانات ثابت أو يضيف/يحذف من النهاية فقط. |
| **القائمة المرتبطة (Linked List)** | كل عنصر (Node) يحمل قيمة ومؤشر (pointer) إلى العنصر التالي. | ```python\nclass Node:\n    def __init__(self,val):\n        self.val=val; self.next=None\nhead = Node(1)\nhead.next = Node(2)\n``` | عندما تحتاج عمليات إدخال/حذف كثيرة في أي موضع (O(1) إذا كان لديك المؤشر). |
| **المكدس (Stack)** | “آخر من يدخل، أول من يخرج” (LIFO). يدعم عمليتي `push` (إضافة) و `pop` (إزالة). | ```python\nstack = []\nstack.append(5)   # push\nstack.pop()       # pop → 5\n``` | لتنفيذ عكوس (مثل تقييم تعبير رياضي)، تتبع الاستدعاءات (call stack)، إلغاء آخر عملية (undo). |
| **الطابور (Queue)** | “أول من يدخل، أول من يخرج” (FIFO). يدعم عمليتي `enqueue` و `dequeue`. | ```python\nfrom collections import deque\nq = deque()\nq.append('a')   # enqueue\nq.popleft()    # dequeue → 'a'\n``` | نمذجة طوابير الانتظار (مصرف، طابور طباعة)، خوارزمية BFS. |
| **الشجرة (Tree)** | بنية هرمية؛ كل عقدة لها أبناء (children). النوع الشائع هو **شجرة البحث الثنائية (Binary Search Tree - BST)**. | ```python\nclass TreeNode:\n    def __init__(self,val):\n        self.val=val; self.left=self.right=None\n``` | البحث السريع (O(log n) في شجرة متوازنة)، تمثيل ملفات النظام، قواعد التعبير. |
| **الرسوم البيانية (Graph)** | مجموعة من العقد (vertices) والحواف (edges) التي تربط بينها. يمكن أن تكون موجهة أو غير موجهة. | ```python\nfrom collections import defaultdict\ngraph = defaultdict(list)\ngraph['A'].append('B')\n``` | نمذجة الشبكات (إنترنت، وسائل التواصل)، خوارزميات أقصر مسار، تدفق. |
| **الجدول التجزيئي (Hash Table / Dictionary)** | تخزين أزواج (مفتاح‑قيمة) مع إمكانية الوصول إلى القيمة عبر المفتاح في زمن ثابت تقريبًا (O(1)). | ```python\nd = {'apple':3, 'banana':5}\nprint(d['apple'])  # 3\n``` | عندما تحتاج بحثًا سريعًا بالمفتاح، كقواميس اللغة، عدّ التكرارات. |
| **كومة (Heap)** | شجرة ثنائية تُحافظ على خاصية “الأصل هو الأكبر (max‑heap) أو الأصغر (min‑heap)”. تُستعمل لتطبيق **Priority Queue**. | ```python\nimport heapq\nheap = []\nheapq.heappush(heap, 5)\nheapq.heappush(heap, 2)\nheapq.heappop(heap)   # 2 (min‑heap)\n``` | جدولة المهام حسب الأولوية، خوارزمية Dijkstra، إيجاد k‑أصغر/أكبر عناصر. |

## 2. الخوارزميات الأساسية

### 2.1 البحث (Search)

| الخوارزمية | المتطلبات | التعقّـيـد | مثال |
|-----------|-----------|-----------|------|
| **البحث الخطي (Linear Search)** | أي مجموعة غير مرتبة. | O(n) | ابحث عن `x` في `[4,2,7,1]` بفحص كل عنصر. |
| **البحث الثنائي (Binary Search)** | مجموعة مرتبة (مصفوفة أو شجرة بحث). | O(log n) | ابحث عن `15` في `[3,8,12,15,19]` بتقسيم النطاق إلى نصفين. |
| **بحث في شجرة (BST Search)** | شجرة بحث ثنائية. | O(h) حيث h هو ارتفاع الشجرة (≈log n إذا كانت متوازنة). | ابحث عن مفتاح `20` بمتابعة الفروع اليسرى/اليمنى. |

### 2.2 الفرز (Sorting)

| الخوارزمية | المميزات | التعقّـيـد (حالة متوسط) | مثال عملي (بايثون) |
|-----------|----------|------------------------|--------------------|
| **الفرز بالدمج (Merge Sort)** | ثابت، يعمل على أي بيانات، يستخدم الذاكرة O(n). | O(n log n) | ```python\ndef merge_sort(a):\n    if len(a) <= 1: return a\n    mid = len(a)//2\n    left, right = merge_sort(a[:mid]), merge_sort(a[mid:])\n    i=j=0; res=[]\n    while i < len(left) and j < len(right):\n        if left[i] < right[j]: res.append(left[i]); i+=1\n        else: res.append(right[j]); j+=1\n    res.extend(left[i:]); res.extend(right[j:]); return res\n``` |
| **الفرز السريع (Quick Sort)** | أسرع في المتوسط، يستخدم مساحة أقل من Merge. | O(n log n) (متوسط) | ```python\nimport random\ndef quick_sort(arr):\n    if len(arr) <= 1: return arr\n    pivot = random.choice(arr)\n    less = [x for x in arr if x < pivot]\n    equal = [x for x in arr if x == pivot]\n    greater = [x for x in arr if x > pivot]\n    return quick_sort(less) + equal + quick_sort(greater)\n``` |
| **الفرز بالاختيار (Selection Sort)** | سهل الفهم، لا يحتاج مساحة إضافية، لكنه بطيء. | O(n²) | ```python\nfor i in range(n):\n    min_idx = i\n    for j in range(i+1, n):\n        if a[j] < a[min_idx]: min_idx = j\n    a[i], a[min_idx] = a[min_idx], a[i]\n``` |

### 2.3 خوارزميات على الرسوم البيانية

| الخوارزمية | ما تحلّه | التعقّـيـد | مثال مختصر |
|-----------|----------|-----------|------------|
| **BFS (Breadth‑First Search)** | أقصر مسار في رسم غير موزون، استكشاف كل الطبقات. | O(V+E) | يستخدم طابور. |
| **DFS (Depth‑First Search)** | اكتشاف المكوّنات المتصلة، كشف دورات. | O(V+E) | يستخدم مكدس (أو استدعاء ذاتي). |
| **Dijkstra** | أقصر مسار في رسم موزون غير سالب. | O((V+E) log V) مع Heap | يدمج جدول تجزيئي لتحديد العقدة ذات المسافة الأقل. |
| **Kruskal / Prim** | شجرة امتداد الحد الأدنى (Minimum Spanning Tree). | O(E log V) | يستخدم جدول تجزيئي لتجنب الدورات. |

### 2.4 خوارزميات أخرى شائعة

| الخوارزمية | الفكرة | الاستخدام |
|-----------|--------|-----------|
| **Two‑Pointer** | مؤشران يتحركان في نفس المصفوفة/السلسلة لتحقيق هدف (مثلاً إيجاد زوج يساوي مجموعًا معينًا). | مشاكل “Two Sum”, حذف العناصر المتكررة. |
| **Sliding Window** | نافذة متحركة بطول ثابت أو متغير لمعالجة أجزاء من البيانات المتتابعة. | إيجاد أطول سلسلة بأكثر من k عناصر مميزة، حساب متوسط متحرك. |
| **Dynamic Programming (DP)** | تقسيم المشكلة إلى حالات فرعية متكررة وتخزين النتائج لتجنب إعادة الحساب. | Fibonacci, Knapsack, Longest Common Subsequence. |
| **Greedy** | اتخاذ قرار محلي “الأفضل الآن” مع أمل أن يكون الحل عالميًا. | Coin Change (عند العملات المتناسقة), Activity Selection. |

## 3. كيف تختار الهيكل والخوارزمية المناسبة؟

1. **ما هو حجم البيانات؟**  
   - إذا كان الحجم صغيرًا (< 10⁴) فغالبًا أي خوارزمية تعمل.  
   - إذا كان كبيرًا (> 10⁶) فانتبه إلى التعقيدات O(n log n) أو O(n).

2. **ما هو نوع العملية الأكثر تكرارًا؟**  
   - كثير من عمليات الإدخال/الحذف → القائمة المرتبطة أو الشجرة المتوازنة.  
   - كثير من عمليات البحث بالـ key → جدول تجزيئي (hash).

3. **هل البيانات مرتبة مسبقًا؟**  
   - إذا نعم → يمكن استخدام البحث الثنائي أو خوارزميات فرز أسرع (مثل insertion sort على قوائم شبه مرتبة).

4. **هل تحتاج إلى الحفاظ على ترتيب معين؟**  
   - المصفوفة تحافظ على الترتيب الأصلي؛ الشجرة تحافظ على ترتيب قيم المفتاح.

5. **الذاكرة المتاحة**  
   - الهياكل التي تحتاج مساحة إضافية (مثل heap أو merge‑sort) قد تكون غير مناسبة على الأجهزة ذات الذاكرة القليلة.

## 4. مثال عملي متكامل: نظام إدارة مكتبة بسيطة

### المتطلبات
1. حفظ الكتب (معرّف، عنوان، مؤلف).  
2. البحث عن كتاب عبر الـ **معرّف** (سرعة عالية).  
3. إظهار جميع الكتب مرتبة حسب العنوان.

### التصميم المقترح
| المتطلب | هيكل البيانات | الخوارزمية |
|--------|----------------|------------|
| حفظ/بحث بالمعرّف | **جدول تجزيئي (Dictionary)** → `id → Book` | عمليات `O(1)` للإضافة والبحث. |
| إظهار مرتّبًا | **قائمة (List)** من الكائنات + **فرز بالدمج** عند الطلب | فرز O(n log n) مرة واحدة عند طلب العرض. |

### كود بسيط (بايثون)

```python
# تعريف كلاس للكتاب
class Book:
    def __init__(self, book_id, title, author):
        self.id = book_id
        self.title = title
        self.author = author
    def __repr__(self):
        return f"{self.id}: {self.title} ({self.author})"

# هياكل البيانات
books_by_id = {}          # hash table
books_list   = []         # لتخزين الترتيب عند الحاجة

# 1- إضافة كتاب
def add_book(book_id, title, author):
    if book_id in books_by_id:
        print("معرّف موجود بالفعل!")
        return
    b = Book(book_id, title, author)
    books_by_id[book_id] = b
    books_list.append(b)

# 2- بحث بالمعرّف
def find_by_id(book_id):
    return books_by_id.get(book_id, "غير موجود")

# 3- عرض جميع الكتب مرتّبة حسب العنوان
def show_sorted():
    # نستخدم merge sort لتوضيح الفكرة (يمكن استدعاء sorted())
    def merge_sort(arr):
        if len(arr) <= 1:
            return arr
        mid = len(arr)//2
        left, right = merge_sort(arr[:mid]), merge_sort(arr[mid:])
        i=j=0; res=[]
        while i < len(left) and j < len(right):
            if left[i].title < right[j].title:
                res.append(left[i]); i+=1
            else:
                res.append(right[j]); j+=1
        res.extend(left[i:]); res.extend(right[j:])
        return res
    sorted_books = merge_sort(books_list)
    for b in sorted_books:
        print(b)

# تجربة
add_book(101, "Python Programming", "Ali")
add_book(102, "Data Structures", "Sara")
add_book(103, "Algorithms", "Omar")
print(find_by_id(102))   # بحث سريع
print("\nالكتب مرتبة حسب العنوان:")
show_sorted()
```

**ما تعلمناه هنا**  
- استخدمنا **جدول تجزيئي** للبحث السريع بالمعرّف.  
- احتفظنا بنسخة من الكتب في **قائمة** لتتمكن من فرزها عندما نحتاج العرض المرتب.  
- استخدمنا **خوارزمية Merge Sort** لتوضيح فكرة الفرز ذات التعقيد `O(n log n)`.

## ٤. أمثلة عملية توضح الفروقات

### مثال 1: حساب مجموع الأعداد من 1 إلى 100

| لغة | كود | زمن التنفيذ (تقريبي على نفس الجهاز) |
|-----|-----|--------------------------------------|
| **Python** | ```python<br>total = sum(range(1,101))<br>print(total)``` | ~0.1 ms (بطيء نسبياً) |
| **C++** | ```cpp<br>#include <iostream><br>int main(){int total=0;for(int i=1;i<=100;i++) total+=i;std::cout<<total;}``` | ~0.02 ms (أسرع) |
| **JavaScript (Node.js)** | ```js<br>let total = 0; for(let i=1;i<=100;i++) total+=i; console.log(total);``` | ~0.05 ms |
| **Rust** | ```rust<br>fn main(){let total: i32 = (1..=100).sum(); println!("{}", total);}``` | ~0.02 ms |

**ما نتعلمه:** للعمليات البسيطة لا تفرق الفروقات كثيراً، لكن عندما تصبح البيانات ضخمة (ملايين، مليارات) يصبح الفرق واضحاً بين اللغات السريعة (C/C++, Rust, Go) واللغات المفسَّرة (Python, JavaScript).

### مثال 2: قراءة ملف نصي وعرض أول 5 أسطر

| لغة | كود |
|-----|-----|
| **Python** | ```python<br>with open('data.txt') as f:<br>    for _ in range(5):<br>        print(f.readline().strip())``` |
| **JavaScript (Node.js)** | ```js<br>const fs = require('fs');<br>const lines = fs.readFileSync('data.txt','utf8').split('\\n');<br>for(let i=0;i<5 && i<lines.length;i++) console.log(lines[i]);``` |
| **Go** | ```go<br>package main<br>import (\"bufio\",\"fmt\",\"os\")<br>func main(){f, _ := os.Open(\"data.txt\"); defer f.Close(); scanner := bufio.NewScanner(f); i:=0; for scanner.Scan() && i<5 { fmt.Println(scanner.Text()); i++ }}``` |
| **Rust** | ```rust<br>use std::fs::File; use std::io::{self, BufRead}; fn main() -> io::Result<()> { let file = File::open(\"data.txt\")?; let reader = io::BufReader::new(file); for (i, line) in reader.lines().enumerate().take(5) { println!(\"{}\", line?); } Ok(()) }``` |

**ما نتعلمه:**  
- **Python** تُظهر أقصر وأكثر وضوحًا بفضل بناء الجملة (syntax) البسيط.  
- **Go** و **Rust** تحتاج إلى مزيد من السطرين لكن تُعطي تحكمًا أكبر في الأخطاء (مثلاً التعامل مع `Result` في Rust).  
- **JavaScript** يبرز سهولة القراءة على الخادم باستخدام مكتبة `fs`.

## Operating systems concepts
## مقدمة بسيطة عن أنظمة التشغيل (Operating Systems)

### ما هو نظام التشغيل؟
نظام التشغيل **هو البرنامج الأساسي** الذي يُشغَّل أولًا عندما تُشغَّل الحاسوب (أو الهاتف الذكي، أو أي جهاز إلكتروني ذكي).  
وظيفته الرئيسية هي **إدارة الموارد** (المعالج، الذاكرة، التخزين، الأجهزة الطرفية) وتوفير واجهة تجعل **المستخدم** والـ **برامج** الأخرى تتفاعل مع الجهاز بسهولة.

> **تشبيه عملي:**  
> تخيَّل أن الحاسوب هو مطعم.  
> - **المعالج (CPU)** هو طباخ الطعام.  
> - **الذاكرة (RAM)** هي طاولة العمل حيث تُحضَّر الأطباق.  
> - **القرص الصلب/SSD** هو مخزن المكونات (الخضار، اللحم، التوابل).  
> - **الأجهزة الطرفية** (مثل الطابعة، الفأرة، الشاشة) هي أدوات الخدمة (المقالي، الأطباق، مناديل).  
> - **نظام التشغيل** هو المدير الذي ينسق كل شيء: يحدد من يطبخ ماذا ومتى، يضمن وجود المكونات الكافية، ويتأكد أن الزبائن (المستخدمين) يحصلون على طلباتهم بسرعة وبطريقة مرتبة.

---

## 1. المهام الأساسية لنظام التشغيل

| المهمة | ماذا تعني؟ | مثال عملي |
|-------|------------|-----------|
| **إدارة العمليات (Process Management)** | إنشاء، جدولة، وإنهاء البرامج التي تعمل (تسمى عمليات). | عندما تفتح متصفح الإنترنت، نظام التشغيل يخلق عملية “Chrome”. إذا فتحت نافذة جديدة، يُنشئ عملية جديدة أو خيط (thread) داخل المتصفح. |
| **إدارة الذاكرة (Memory Management)** | تخصيص الذاكرة للبرامج، ومنع تداخلها، وإعادة تدوير الذاكرة غير المستخدمة. | إذا كان لديك 8 GB RAM، قد يستخدم نظام التشغيل 2 GB لتشغيل نظام الويندوز، 1 GB لتشغيل برنامج تحرير النصوص، والباقي يظل متاحًا لتطبيقات أخرى. |
| **إدارة التخزين (File System)** | تنظيم الملفات داخل الأقراص الصلبة/الـ SSD وتوفير طرق للوصول إليها. | عندما تحفظ صورة في مجلد “صور” على سطح المكتب، نظام التشغيل يضعها في بنية ملفات (مثل NTFS أو ext4) ويعطيها مسارًا مثل `C:\Users\Ahmed\Pictures\photo.jpg`. |
| **إدارة الأجهزة (Device Management)** | توفير برامج تشغيل (Drivers) لتواصل البرامج مع الأجهزة (طابعات، بطاقات شبكة، إلخ). | عند توصيل USB‑Flash، يكتشف نظام التشغيل الجهاز، يحمّله (install driver) ويجعله جاهزًا للنسخ واللصق. |
| **الأمان والتحكم (Security & Access Control)** | حماية الموارد من الاستخدام غير المصرح به وتحديد صلاحيات المستخدمين. | حساب مستخدم "Guest" لا يستطيع حذف ملفات النظام، بينما حساب "Administrator" يملك صلاحيات كاملة. |
| **واجهة المستخدم (User Interface)** | تقديم واجهة رسومية (GUI) أو سطر أوامر (CLI) لتفاعل المستخدم مع النظام. | Windows Explorer، macOS Finder، أو سطر الأوامر في Linux (`bash`). |

---

## 2. المفاهيم الأساسية

### 2.1 العملية (Process) والخيط (Thread)
- **العملية**: برنامج قيد التنفيذ مع مساحة الذاكرة الخاصة به. مثال: تشغيل برنامج Microsoft Word هو عملية.
- **الخيط**: وحدة أصغر داخل العملية تستطيع تنفيذ جزء من التعليمات بصورة متوازية. مثال: داخل Word قد يكون هناك خيط لفتح الملف وآخر لحفظ التغييرات تلقائيًا.

### 2.2 جدولة (Scheduling)
نظام التشغيل يقرر **متى** سيعمل كل معالج (CPU) على أي عملية أو خيط.  
- **خوارزميات جدولة شائعة**:  
  - **Round‑Robin**: يخصص كل عملية شريحة زمنية (time‑slice) متساوية، ثم ينتقل للتي تليها. مفيد في الأنظمة التفاعلية.  
  - **Priority Scheduling**: تُعطى عمليات ذات أولوية أعلى وقتًا أكبر على المعالج.  

### 2.3 الذاكرة الافتراضية (Virtual Memory)
تسمح للبرامج بالوصول إلى مساحة ذاكرة أكبر من الـ RAM الفعلي عن طريق **الصفحات (pages)** التي تُخزن جزئيًا على القرص الصلب.  
- **مثال عملي:** إذا كان لديك 4 GB RAM فقط، لكن برنامج تحرير فيديو يحتاج 6 GB، فإن نظام التشغيل يستخدم جزءًا من القرص كـ “صفحة تبادل” (swap/page file) لتخزين البيانات غير النشطة مؤقتًا.

### 2.4 نظام الملفات (File System)
يُعرِّف كيفية تخزين البيانات واسترجاعها.  
- **NTFS** (Windows) يدعم الأمان المتقدم والملفات الكبيرة.  
- **ext4** (Linux) سريع ومستقر.  
- **APFS** (macOS) يوفّر نسخًا احتياطية فورية (snapshots).

### 2.5 إدارة الأجهزة (Device Drivers)
كل جهاز يحتاج برنامجًا خاصًا (driver) يترجم الأوامر العامة لنظام التشغيل إلى أوامر يفهمها الجهاز.  
- **مثال:** عند طباعة مستند، برنامج الطباعة يرسل أوامر إلى برنامج تشغيل الطابعة، والذي يحوّلها إلى إشارات كهربائية تُرسل إلى رأس الطباعة.

### 2.6 الأمان (Security)
- **المصادقة (Authentication):** التأكد من هوية المستخدم (كلمة مرور، بصمة، بطاقة ذكية).  
- **التفويض (Authorization):** تحديد ما يمكن للمستخدم فعله (قراءة ملف، تعديل، حذف).  
- **العزل (Isolation):** كل عملية تعمل في مساحة منفصلة لمنع “التسرب” بين البرامج (مثلاً، تعطل برنامج لا يتسبب في تعطل النظام ككل).

## 4. تجربة عملية (Hands‑On) بسيطة

### مثال 1: مشاهدة جدولة العمليات في Linux
1. **افتح الطرفية (Terminal).**  
2. اكتب الأمر `top` ثم اضغط Enter.  
3. سترى قائمة بالعمليات الجارية، استهلاك الـ CPU، والـ RAM.  
4. اضغط `q` للخروج.

> **ما يحدث خلف الكواليس؟**  
> نظام التشغيل يقرأ جدول العمليات (process table) ويظهر لك أهم المعلومات. الـ **kernel** هو قلب النظام الذي يدير هذه العملية.

### مثال 2: إنشاء ملف وحذفّه في Windows باستخدام سطر الأوامر
1. افتح **Command Prompt** (ابدأ → تشغيل → `cmd`).  
2. اكتب:  
   ```cmd
   echo Hello World > C:\Temp\test.txt
   type C:\Temp\test.txt
   del C:\Temp\test.txt
   ```  
3. ستظهر لك كلمة “Hello World”، ثم يُحذف الملف.

> **ما يحدث؟**  
> - `echo` يرسل النص إلى نظام الملفات.  
> - `>` يوجه الخرج إلى ملف جديد (`test.txt`).  
> - `type` يقرأ الملف ويعرض محتواه.  
> - `del` يطلب من نظام التشغيل حذف الكائن من نظام الملفات.

### مثال 3: تشغيل برنامج في الخلفية (Background) على macOS / Linux
```bash
sleep 30 &
```
- الأمر `sleep 30` يوقف التنفيذ 30 ثانية.  
- إشارة `&` تجعل العملية تعمل في الخلفية، لذا يمكنك الاستمرار في كتابة أوامر أخرى.  
- استخدم `jobs` لعرض الوظائف الخلفية، و`kill %1` لإيقاف الأولى.

## 5. لماذا تُعدّ أنظمة التشغيل مهمة لل مبرمج؟

1. **فهم الموارد:** معرفة كيف تُدار الذاكرة والـ CPU يساعدك على كتابة برامج أكثر كفاءة (مثلاً تجنّب تسريبات الذاكرة).  
2. **التعامل مع الملفات:** كل برنامج يحتاج يقرأ/يكتب بيانات؛ معرفة نظام الملفات يسهّل إدارة المسارات واختيار الصيغ المناسبة.  
3. **الأمان:** إذا عرفت كيف يتحقق النظام من الصلاحيات، يمكنك تجنّب الثغرات (مثل الـ *privilege escalation*).  
4. **التنفيذ المتوازي:** الفهم الجيد للـ threads والـ processes يتيح لك استغلال المعالجات المتعددة (multi‑core) لتسريع التطبيقات.

## 6. خلاصة سريعة (Cheat‑Sheet)

| مفهوم | ما هو؟ | مثال |
|------|--------|------|
| **Kernel** | نواة النظام، تدير العتاد والموارد | جدولة الـ CPU، إدارة الذاكرة |
| **Process** | برنامج يعمل مع مساحة ذاكرة خاصة | تشغيل Spotify |
| **Thread** | جزء من العملية يعمل بشكل مستقل | خيط تشغيل الموسيقى داخل Spotify |
| **Virtual Memory** | توسيع RAM باستخدام مساحة على القرص | الـ swap في Linux |
| **File System** | طريقة تنظيم وتخزين الملفات | NTFS, ext4, APFS |
| **Driver** | برنامج يترجم أوامر OS إلى الجهاز | برنامج تشغيل بطاقة الشبكة |
| **User Mode vs Kernel Mode** | وضع المستخدم لا يملك صلاحيات عالية؛ وضع النواة يملكها | تطبيقات تعمل في user mode، النظام في kernel mode |
| **Scheduling** | كيفية توزيع وقت الـ CPU على العمليات | Round‑Robin, Priority |
| **Security** | المصادقة، التفويض، العزل | حسابات المستخدمين، صلاحيات الملفات |

### الآن أنت جاهز للخطوة التالية!
- **استكشاف**: جرّب أوامر `top`, `ps`, `htop` على نظام Linux أو macOS؛ أو `Task Manager` على Windows.  
- **القراءة**: ابحث عن مقالات حول “process scheduling algorithms” أو “virtual memory paging”.  
- **التجربة**: اكتب برنامج بسيط (مثلاً بلغة Python) يطبع “Hello” ثم ينام 5 ثوانٍ، وشغّله مرتين لتلاحظ عمليتي الـ **process** و**thread** في `top`.

بهذا تكون قد غطيت الأساسيات التي يحتاجها أي مبتدئ لفهم أنظمة التشغيل، مع أمثلة واقعية تجعل المفاهيم أكثر وضوحًا. إذا كان لديك سؤال محدد (مثلاً عن نظام ملفات معين أو عن كيفية كتابة driver)، أخبرني وسأشرح لك بالتفصيل!


## Computer networks basics
## مقدمة بسيطة عن شبكات الحاسوب  

**شبكة الحاسوب** هي مجموعة من الأجهزة (حاسوب، هاتف، طابعة، خادم… إلخ) مرتبطة ببعضها لتبادل المعلومات.  
تخيل أن الأجهزة هي أشخاص في مدينة، والشبكة هي شبكة الطرق التي تسمح لهم بالتواصل وإرسال الرسائل أو نقل البضائع.

---

## 1. لماذا نحتاج شبكة؟

| الحاجة | مثال عملي |
|--------|-----------|
| مشاركة الملفات | فريق عمل يرفع مستندات على خادم مركزي ويستطيع الجميع تحميلها. |
| مشاركة الطابعات | طابعة واحدة في المكتب تُستخدم من قبل جميع الحواسيب. |
| الوصول إلى الإنترنت | كل الحواسيب في المنزل أو الشركة تستخدم اتصالاً واحداً للإنترنت. |
| الألعاب الجماعية | لاعبون من دول مختلفة يتصلون بخادم اللعبة لتلعب معاً. |
| الخدمات السحابية | تخزين الصور على جوجل درايف أو مشاهدة فيديوهات على يوتيوب. |

---

## 2. مكونات الشبكة الأساسية

| المكوّن | وظيفته | مثال |
|--------|--------|------|
| **الأجهزة الطرفية (End Devices)** | هي الأجهزة التي يستخدمها الأشخاص. | حاسوب مكتبي، لابتوب، هاتف ذكي، طابعة. |
| **وسائط النقل (Transmission Media)** | الوسيلة التي تنتقل عليها البيانات. | كابل إيثرنت (Copper), ألياف بصرية (Fiber), موجات الراديو (Wi‑Fi, Bluetooth). |
| **الموجهات (Routers)** | توجيه البيانات بين شبكات مختلفة (مثلاً بين شبكة المنزل والإنترنت). | الموجه (Router) الذي يقدمه مزوّد الخدمة. |
| **المفاتيح (Switches)** | ربط العديد من الأجهزة داخل نفس الشبكة المحلية (LAN). | مفتاح إيثرنت في مكتب يربط 10 حواسيب. |
| **نقطة الدخول (Gateway)** | جسر بين شبكتين تستخدمان بروتوكولات مختلفة. غالبًا يكون الموجه نفسه هو الـ Gateway للإنترنت. |
| **البروتوكولات (Protocols)** | قواعد “اللغة” التي تتبعها الأجهزة لتفهم بعضها البعض. | TCP/IP، HTTP، FTP، DNS. |

---

## 3. المصطلحات الشائعة

| المصطلح | المعنى (بسيط) |
|---------|--------------|
| **LAN (Local Area Network)** | شبكة محلية تغطي مساحة صغيرة (منزل، مكتب). |
| **WAN (Wide Area Network)** | شبكة تغطي مسافات واسعة (الإنترنت هو أكبر WAN). |
| **IP Address** | عنوان يحدد كل جهاز على الشبكة، مثل رقم منزل. مثال: `192.168.1.10`. |
| **MAC Address** | عنوان فريد للبطاقة الشبكية داخل الجهاز، لا يتغير. |
| **Subnet Mask** | يحدد أي جزء من الـ IP هو “المنطقة” (الشبكة) وأي جزء هو “الهوست” (الجهاز). |
| **DHCP** | خدمة تعطي الأجهزة عناوين IP تلقائيًا. |
| **DNS** | نظام أسماء النطاقات؛ يترجم `www.google.com` إلى عنوان IP. |
| **Ping** | أداة بسيطة لاختبار ما إذا كان جهاز آخر متصلًا. |
| **Firewall** | جدار حماية يراقب ويُفلتر حركة البيانات. |

---

## 4. كيف تُنشئ شبكة بسيطة في المنزل؟

1. **المكونات المطلوبة**  
   - موفر إنترنت (ISP) يزودك بخط (DSL, فايبر، كابل).  
   - موجه (Router) يدعم Wi‑Fi.  
   - كابل إيثرنت (اختياري لتوصيل الأجهزة مباشرة).  

2. **خطوات الإعداد**  
   1. **اتصل بالخط**: يربط موفر الخدمة (مثلاً كابل) بالـ WAN Port في الموجه.  
   2. **تشغيل الموجه**: بعد توصيل الكهرباء، سيتعامل الموجه مع مزود الخدمة ويُعطيك عنوان IP خارجي.  
   3. **تكوين شبكة Wi‑Fi**: عبر الدخول إلى واجهة الويب للموجه (عادة `192.168.0.1` أو `192.168.1.1`) وتحديد اسم الشبكة (SSID) وكلمة مرور.  
   4. **الاتصال بالأجهزة**:  
      - على اللابتوب أو الهاتف، ابحث عن اسم الـ Wi‑Fi واختره، ثم أدخل كلمة المرور.  
      - إذا أردت سرعة أعلى أو أمان أكثر، وصل جهازك بالكابل إلى أحد الـ LAN Ports في الموجه.  

3. **اختبار الاتصال**  
   - افتح موجه الأوامر (Command Prompt) واكتب: `ping google.com`.  
   - إذا ظهر “Reply from …”, فالاتصال يعمل.

---

## 5. مثال عملي: مشاركة ملف بين حاسوبين في نفس الغرفة  

1. **التحقق من الاتصال**  
   - تأكد أن الحاسوبين متصلان بنفس الـ Wi‑Fi أو كلاهما موصول بكابل إلى نفس المفتاح (Switch).  
2. **تفعيل مشاركة الملفات (Windows)**  
   - افتح “Network and Sharing Center” → “Change advanced sharing settings”. فعل “Turn on network discovery” و “Turn on file and printer sharing”.  
3. **إنشاء مجلد مشترك**  
   - انقر بزر الفأرة الأيمن على مجلد → “Properties” → “Sharing” → “Advanced Sharing” → اختر “Share this folder”.  
4. **الوصول إلى المجلد من الحاسوب الثاني**  
   - افتح “File Explorer” واكتب في شريط العنوان: `\\192.168.1.5\اسم_المجلد` (استبدل الـ IP بالعنوان الفعلي للجهاز الأول).  
5. **نقل الملف**  
   - اسحب الملف إلى المجلد المشترك أو العكس.

## 6. مفهوم الطبقات (OSI Model) بصورة مبسطة  

| الطبقة | ما تفعله (مثال) |
|--------|----------------|
| **7. التطبيق (Application)** | متصفح الويب يرسل طلب صفحة. |
| **6. العرض (Presentation)** | تشفير البيانات (HTTPS). |
| **5. الجلسة (Session)** | إنشاء جلسة اتصال بين المتصفح والخادم. |
| **4. النقل (Transport)** | تقسيم البيانات إلى أجزاء (TCP) وإعادة تجميعها. |
| **3. الشبكة (Network)** | اختيار الطريق (IP) لتوجيه الحزم. |
| **2. ربط البيانات (Data Link)** | إضافة عنوان MAC وإرسال الإطارات داخل نفس الشبكة. |
| **1. الفيزيائية (Physical)** | تحويل البتات إلى إشارات كهربائية أو ضوئية عبر الكابل. |

نحن لا نحتاج أن نبرمج كل طبقة، لكن فهم الفكرة يساعد عند حدوث مشاكل (مثلاً مشكلة في الـ IP = طبقة 3، مشكلة في كابل الإيثرنت = طبقة 1).

---

## 7. أمثلة على مشاكل شائعة وطرق حلها

| المشكلة | السبب المحتمل | طريقة الفحص/الحل |
|--------|---------------|-------------------|
| لا أستطيع الاتصال بالإنترنت | الموجه غير متصل بالـ ISP | افحص أضواء WAN في الموجه؛ أعد تشغيل الموجه. |
| لا أرى الأجهزة الأخرى في الشبكة | عنوان IP غير في نفس النطاق | تحقق من الـ IP (`ipconfig` على Windows) وتأكد من أنه مثل `192.168.1.x`. |
| بطء شديد في تحميل الفيديوهات | ازدحام Wi‑Fi أو إشارة ضعيفة | قرب الجهاز من الموجه أو استخدم كابل إيثرنت؛ غير قناة الـ Wi‑Fi. |
| لا يمكن طباعة ملف على الطابعة الشبكية | جدار حماية يمنع الاتصالات | أضف استثناء للطابعة في إعدادات الـ Firewall. |
| رسالة “Destination Host Unreachable” عند ping | لا يوجد مسار إلى الجهاز الهدف | تحقق من أن الجهاز الهدف شغال ومتصّل، أو أن الـ subnet mask متطابق. |

## 8. خطوات لتعلم أكثر (مسار عملي)

1. **الأساسيات**  
   - افهم ما هو الـ IP, MAC, DNS, DHCP.  
   - جرب أوامر سطر الأوامر: `ipconfig /all`, `ping`, `tracert`, `nslookup`.  

2. **التحكم بالموجه**  
   - ادخل إلى واجهة الموجه، استكشف إعدادات WAN, LAN, DHCP, Port Forwarding.  

3. **بناء شبكة صغيرة**  
   - احصل على مفتاح (Switch) وموجه، صِل 2–3 حواسيب، جرّب مشاركة الملفات والطباعة.  

4. **التعمق في البروتوكولات**  
   - اقرأ عن TCP vs UDP، كيف يعمل HTTP/HTTPS، ما هو TLS.  

5. **الأمن**  
   - فعل جدار حماية، غير كلمة مرور الموجه، استخدم تشفير WPA3 للـ Wi‑Fi.  

6. **مشاريع تطبيقية**  
   - **خادم ويب بسيط**: نصب Apache أو Nginx على لابتوب، وجرب الوصول إليه من جهاز آخر.  
   - **شبكة VPN**: استخدم OpenVPN أو WireGuard لإنشاء اتصال آمن بين جهازين عبر الإنترنت.

## خلاصة

- **شبكة الحاسوب** هي وسيلة لتوصيل الأجهزة وتبادل البيانات.  
- **المكونات الأساسية**: الأجهزة الطرفية، وسائط النقل، الموجهات، المفاتيح، والبروتوكولات.  
- **البدء عمليًا**: استخدم الموجه المنزلي لتنشئ شبكة Wi‑Fi، جرب مشاركة الملفات، واختبر الاتصال بأوامر بسيطة.

بهذه الخطوات، يمكنك الانتقال من طالب جديد إلى شخص قادر على بناء وإدارة شبكة بسيطة، ثم توسيع معرفتك لتشمل شبكات أكبر وأكثر أمانًا. بالتوفيق! 🚀


## Software engineering principles
## 📚 مقدمة سريعة  
**هندسة البرمجيات (Software Engineering)** هي مجموعة من المبادئ، الأساليب، والأدوات التي تساعدنا على بناء برامج ذات جودة عالية، سهلة الصيانة، وقابلة للتوسع.  
تخيل أنك تريد بناء منزل: لا يمكنك أن تبدأ بوضع الطوب عشوائياً؛ تحتاج مخطط، مواد، فريق عمل، ومراحل واضحة. نفس الفكرة تنطبق على تطوير البرمجيات.

---

## 1️⃣ مبدأ **الـ SOLID** (أساسيات التصميم الكائني)

| مبدأ | معنى مبسط | مثال عملي |
|------|-----------|-----------|
| **S** – **Single Responsibility Principle** (مبدأ المسؤولية الواحدة) | كل وحدة (صف، دالة، مكوّن) يجب أن تقوم بعمل واحد فقط. | *دالة* `calculateInvoiceTotal()` لا يجب أن تقوم بطباعة الفاتورة. اجعل عملية الحساب في دالة، وطباعة الفاتورة في دالة أخرى. |
| **O** – **Open/Closed Principle** (مبدأ الفتح والإغلاق) | الكود يجب أن يكون **مفتوحًا** للامتداد **مغلقًا** للتعديل. | إذا أردت إضافة طريقة دفع جديدة، أضف فئة `PaymentMethod` جديدة بدلاً من تعديل الفئة الموجودة `Checkout`. |
| **L** – **Liskov Substitution Principle** (مبدأ استبدال لسكوف) | كل فئة فرعية يجب أن تكون قادرة على استبدال الفئة الأصلية دون كسر السلوك. | إذا كان لديك `Bird` مع طريقة `fly()`, لا تجعل فئة `Penguin` ترث منها وتعيد تنفيذ `fly()` لتفعل شيئًا مختلفًا (البطاريق لا تطير). |
| **I** – **Interface Segregation Principle** (مبدأ تجزئة الواجهة) | لا تجبر الكائنات على تنفيذ واجهات لا تحتاجها. | بدلاً من واجهة `Worker` التي تحتوي على `code()`, `design()`, `test()`, اصنع واجهات منفصلة `Coder`, `Designer`, `Tester`. |
| **D** – **Dependency Inversion Principle** (مبدأ عكس الاعتمادية) | اعتمد على تجريدات (واجهات/عقود) وليس على تنفيذات concrete. | في تطبيق طلبات الطعام، بدلاً من أن `OrderService` يخلق مباشرةً `SqlOrderRepository`, اجعلها تعتمد على واجهة `IOrderRepository`، ثم قم بحقن تنفيذ `SqlOrderRepository` أو `InMemoryOrderRepository` حسب الحاجة. |

**لماذا SOLID مهم؟**  
- يقلل التعقيد.  
- يجعل الكود أكثر قابلية لإعادة الاستخدام.  
- يسهل اختبار الوحدات (unit testing).

## 2️⃣ مبدأ **DRY** – **Don’t Repeat Yourself** (لا تكرّر نفسك)

**الفكرة:** إذا وجدت نفس الكود أو المنطق يتكرر في أكثر من موضع، اجعله وحدة واحدة (دالة/كلاس) تُستدعى كلما احتجت إليه.

### مثال عملي
```python
# كود مكرر
price = 100
tax = price * 0.15
total = price + tax
print(total)

# مرة أخرى في ملف آخر
price = 250
tax = price * 0.15
total = price + tax
print(total)
```

**الحل (DRY):**
```python
def calculate_total(price):
    tax = price * 0.15
    return price + tax

print(calculate_total(100))
print(calculate_total(250))
```

**الفائدة:** إذا تغيرت نسبة الضريبة من 15% إلى 18%، نعدل الكود في مكان واحد فقط.

---

## 3️⃣ مبدأ **KISS** – **Keep It Simple, Stupid** (اجعلها بسيطة)

- لا تكتب حلولاً معقدة إذا كان هناك حل أبسط.
- استخدم أسماء واضحة، تجنّب التعقيدات غير الضرورية.

### مثال عملي
بدلاً من كتابة خوارزمية فرز معقدة يدويًا، استخدم المكتبة المدمجة:

```java
// معقد
for (int i = 0; i < arr.length; i++) {
    for (int j = i+1; j < arr.length; j++) {
        if (arr[i] > arr[j]) {
            int tmp = arr[i];
            arr[i] = arr[j];
            arr[j] = tmp;
        }
    }
}

// بسيط
Arrays.sort(arr);
```

---

## 4️⃣ مبدأ **YAGNI** – **You Aren’t Gonna Need It** (أنت لا تحتاجه الآن)

- لا تُضيف ميزات أو تعقيدات “للمستقبل” إذا لم تكن مطلوبة الآن.
- كل ميزة إضافية تزيد من تكلفة الصيانة والاختبار.

### مثال عملي
أنت تبني تطبيق ملاحظات. لا تضيف الآن نظامًا معقدًا للتصنيف الهرمي أو دمج الذكاء الاصطناعي إذا كان المطلوب فقط إنشاء / تعديل / حذف ملاحظة. أضف هذه الميزات فقط عندما يطلبها المستخدمون فعليًا.

---

## 5️⃣ مبدأ **Separation of Concerns** (فصل الاهتمامات)

- قسّم النظام إلى طبقات أو مكونات بحيث كل طبقة تهتم بجزء واحد من الوظيفة العامة.
- يُسهّل ذلك الصيانة والاختبار.

### مثال عملي (معمارية ثلاثية الطبقات)

| طبقة | مسؤولياتها |
|------|------------|
| **Presentation (UI)** | عرض البيانات للمستخدم، جمع الإدخالات. |
| **Business Logic** | القواعد التجارية، المعالجة. |
| **Data Access** | قراءة/كتابة البيانات من قاعدة البيانات أو ملفات. |

```text
[UI] <---> [Service] <---> [Repository] <---> [Database]
```

إذا تغيرت طريقة تخزين البيانات (مثلاً من MySQL إلى MongoDB)، نحتاج لتعديل طبقة الـ Repository فقط، دون التأثير على الـ UI أو الـ Service.

---

## 6️⃣ مبدأ **Test‑Driven Development (TDD)** (تطوير بقيادة الاختبار)

1. **اكتب اختبارًا فشل** (Red).  
2. **اكتب الحد الأدنى من الكود** لجعل الاختبار ينجح (Green).  
3. **حسّن الكود** (Refactor) مع الحفاظ على نجاح الاختبار.

### مثال عملي (بايثون)

```python
import unittest

# 1️⃣ اختبار فشل
class TestMath(unittest.TestCase):
    def test_add(self):
        self.assertEqual(add(2, 3), 5)

if __name__ == '__main__':
    unittest.main()
```

تشغيل الاختبار سيعطي خطأ لأن `add` غير موجود. الآن نضيف الدالة:

```python
def add(a, b):
    return a + b
```

تشغيل الاختبار مرة أخرى → **نجاح**. ثم يمكن تحسين الدالة إذا احتجنا (مثلاً إضافة تحقق من الأنواع) مع الحفاظ على نجاح الاختبار.

**الفائدة:** يضمن وجود تغطية اختبارية، يقلل الأخطاء عند تعديل الكود.

---

## 7️⃣ مبدأ **Continuous Integration / Continuous Delivery (CI/CD)**

- **CI**: دمج الكود في المستودع الرئيسي بصورة متكررة (عادةً كل 5‑10 دقائق) مع تشغيل اختبارات تلقائية.
- **CD**: نشر النسخة التي اجتازت جميع الاختبارات إلى بيئة الاختبار أو الإنتاج تلقائيًا.

### مثال عملي
- **GitHub Actions** أو **GitLab CI** تقوم بتشغيل:
  1. `npm install` (تثبيت الحزم)  
  2. `npm test` (تشغيل اختبارات الوحدة)  
  3. `npm run build` (إنشاء نسخة الإنتاج)  
  4. نشر إلى **Heroku** أو **AWS Elastic Beanstalk** إذا نجحت جميع الخطوات.

بهذا، كل مرة يضيف أحدهم ميزةً أو يصحح خطأً، يتم التأكد تلقائيًا من عدم كسر شيء آخر.

---

## 8️⃣ مبدأ **الـ “Big‑O”** (تحليل التعقّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّـّ​  
- **CI** يضمن أن كل تعديل يمر عبر اختبارات الوحدة، التكامل، وربما اختبارات الأداء.  
- **CD** يضمن أن النسخة الصالحة تصل إلى بيئة الاختبار أو الإنتاج بسرعة، مما يقلل الفجوة بين التطوير والتشغيل.

## 9️⃣ مبدأ **الـ “YAGNI”** (مكرر) – تم تغطيته، لكن تذكّر دائمًا

## 📦 خلاصة سريعة للطالب الجديد

| مبدأ | لماذا يهم؟ | مثال عملي |
|------|-----------|-----------|
| **SOLID** | بنية كائنات نظيفة، سهل الصيانة | فصل مسؤولية حساب الفاتورة عن طباعتها |
| **DRY** | تجنّب تكرار الكود → صيانة أسهل | دالة `calculate_total(price)` |
| **KISS** | الحلول البسيطة أقل أخطاء | استخدم `Arrays.sort` بدل كتابة فرز يدوي |
| **YAGNI** | لا تضف ميزات غير مطلوبة | لا تبني نظام توصية قبل طلب العملاء |
| **Separation of Concerns** | كل جزء يركز على شيء واحد | طبقة UI، طبقة Service، طبقة Repository |
| **TDD** | كود موثوق؛ الأخطاء تُكتشف مبكراً | كتابة اختبار `add()` قبل تنفيذ الدالة |
| **CI/CD** | دمج مستمر، نشر سريع، جودة ثابتة | GitHub Actions + Heroku للنشر الآلي |
| **Big‑O** | قياس كفاءة الخوارزميات | خوارزمية فرز O(n log n) vs O(n²) |

## 🛠️ خطوات تطبيق هذه المبادئ في مشروعك الأول

1. **ابدأ بتصميم بسيط** (KISS). ارسم مخططًا للطبقات (UI‑Service‑Repository).  
2. **اكتب اختبارات الوحدة** (TDD) قبل كل دالة رئيسية.  
3. **طبق SOLID** عند إنشاء الكلاسات؛ أنشئ واجهات إذا كنت تتوقع توسيع المستقبل.  
4. **احرص على DRY** – احذف أي كود مكرر فوراً.  
5. **استخدم أداة CI** (GitHub Actions) لتشغيل الاختبارات تلقائيًا عند كل `push`.  
6. **لا تضف ميزات غير مطلوبة** (YAGNI). استقبل طلبات المستخدمين ثم نفذها واحدةً تلو الأخرى.  
7. **راقب الأداء** باستخدام تحليل الـ Big‑O إذا لاحظت بطءً في جزء ما.  

باتباع هذه الخطوات، ستحصل على برنامج **قابل للصيانة، قابل للتوسع، ومُختبر جيدًا**، وهو ما يطمح إليه كل مهندس برمجيات محترف. 🚀

---

**ملاحظة أخيرة:** لا تتوقع إتقان كل هذه المبادئ في أسبوع واحد. ابدأ بتطبيق مبدأ واحد (مثلاً DRY) ثم أضف أخرى تدريجيًا مع كل مشروع جديد. التجربة العملية هي أفضل معلم! 🎓
